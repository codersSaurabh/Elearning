Immutable Data Types in Python

In Python, an immutable data type is a type of object whose value cannot be changed after it is created.
Once an immutable object is created, it cannot be modified. Instead, if you need to change the value,
you create a new object. This is in contrast to mutable data types, which can be modified after creation.

Examples of common immutable data types in Python include:

l>
  • int: Integer values are immutable. If you want to change an integer, you create a new integer.
  • float: Floating-point numbers are also immutable.
  • str: Strings are immutable. To change a string, you create a new one.
  • tuple: Tuples are ordered collections of items and are immutable. Once a tuple is created, you cannot change its elements.

    • It's important to understand the difference between mutable and immutable types in Python, as it can have
    implications for how your code behaves and helps you avoid unexpected side effects when working with data.
